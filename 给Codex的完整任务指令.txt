## 一、开始修改前必须先做的事情

1. 读取项目全部主要文件，确认技术栈、页面入口、路由、模板、CSS、JavaScript、Cloudflare Worker 入口、API、数据存储和部署方式。
2. 确认 `/admin`、`/admin/config` 以及添加、编辑订阅页面分别由哪些文件生成。
3. 检查是否已有 `manifest`、Service Worker、图标、离线缓存和 Apple Web App 配置。
4. 先在回复中列出：
   - 当前项目结构摘要
   - 准备修改的文件
   - 准备新增的文件
   - 可能影响现有业务的风险
5. 随后直接实施，不要只给建议。

## 二、核心原则

1. 保留现有订阅管理、提醒计算、登录、退出、通知渠道、API、Cron、数据库或 KV/D1 逻辑。
2. 不要随意修改后端字段名、接口地址和数据格式。
3. 不要重写整个后端。
4. 优先做最小侵入式前端重构。
5. 不得把密码、Token、API Key、Webhook、Cloudflare 凭据写进前端或提交到仓库。
6. 页面名称统一显示为“别忘订阅”。
7. 可以在代码内部保留原项目名，避免破坏部署配置，但用户可见区域统一使用“别忘订阅”。
8. 图标尽量使用项目已有图标库或统一的 SVG 图标，不要使用风格不一致的 Emoji。

## 三、电脑端页面要求

### 1. 顶部导航

- 使用紧凑白色导航栏。
- 左侧显示应用图标、名称“别忘订阅”和简短说明。
- 右侧保留订阅列表、系统配置、退出登录。
- 当前页面使用浅蓝背景或描边高亮。
- 时间信息弱化处理，不要占据过大空间。
- 页面最大内容宽度保持一致，建议约 1120 至 1280 像素。

### 2. 订阅首页 `/admin`

按照 `desktop_dashboard.png` 优化：

- 删除现有过高、视觉过重的大面积渐变横幅。
- 顶部改成简洁的标题区和统计卡片。
- 保留全部订阅、启用中、即将到期、已过期统计。
- 统计卡片高度统一，数字突出，说明文字弱化。
- 搜索、分类、农历开关和添加订阅放在同一功能区。
- 添加按钮保持醒目，但不要过宽。
- 表格列对齐，空状态居中。
- 小屏电脑和平板可以将表格转换为卡片列表，避免横向挤压。
- 保留原来的排序、筛选和操作功能。

### 3. 添加和编辑订阅

按照：
- `desktop_add_subscription.png`
- `desktop_edit_subscription.png`

要求：

- 桌面端可使用居中弹窗或独立卡片。
- 表单按逻辑分组，减少一整页密集输入框。
- 名称、分类、类型、价格、付款周期、到期日期、提醒规则等字段必须保留。
- 主要按钮和取消按钮层级清晰。
- 表单错误显示在对应字段附近。
- 保存时显示加载状态，防止重复提交。

### 4. 系统配置 `/admin/config`

按照 `desktop_settings.png` 优化：

- 不再把所有配置无间隔地堆成超长页面。
- 分成：账户、显示、时区、提醒、通知渠道、安全与 API。
- 可以使用左侧目录、顶部标签页或折叠分组。
- 只有勾选的通知方式才展开对应配置。
- Telegram、NotifyX、Webhook、企业微信、邮件、Bark 等配置保持原功能。
- 测试通知按钮放在对应渠道卡片内部。
- 页面底部或右下角增加明确的“保存设置”按钮。
- 长页面保存按钮可做粘性定位，但不能遮挡内容。

## 四、手机端页面要求

使用真正的手机响应式布局，不要简单把电脑页面按比例缩小。

### 1. 通用规则

- 支持 360、375、390、412、430 像素宽度。
- 禁止出现横向滚动。
- 输入框、选择框和按钮触控高度建议不低于 44 像素。
- 兼容刘海屏和底部安全区域：
  `env(safe-area-inset-top)`、`env(safe-area-inset-bottom)`。
- 底部导航固定时必须给内容区域留足下边距。
- 字号保持清楚，不使用过小文字。
- 弹窗在手机端改为底部抽屉或全屏表单。

### 2. 手机首页

按照 `mobile_home.png`：

- 顶部显示“别忘订阅”、提醒概况和用户入口。
- 用小型统计卡片展示当前状态。
- 优先展示即将到期的订阅。
- 首页不展示过多管理配置。
- 使用简洁卡片，每张订阅卡显示名称、到期时间、状态和价格。

### 3. 手机订阅列表

按照 `mobile_subscriptions.png`：

- 搜索框置顶。
- 分类筛选使用横向滚动标签或下拉菜单。
- 订阅使用卡片列表，不使用宽表格。
- 操作菜单收进每张卡片右上角的更多按钮。
- 底部导航包含：首页、订阅、添加、设置。
- 中间添加按钮是正方形或圆角正方形，不要做成长条按钮。

### 4. 手机添加订阅

按照 `mobile_add_subscription.png`：

- 使用单列布局。
- 将表单拆成基础信息、计费信息、到期与提醒。
- 底部保存按钮可固定，但必须兼容安全区域。
- 日期选择和数字输入适配手机键盘。
- 保存失败时保留用户已填写内容。

### 5. 手机设置

按照 `mobile_settings.png`：

- 设置项用列表和分组卡片呈现。
- 通知渠道可进入二级页面或展开配置。
- 敏感信息默认遮挡。
- 保留测试通知功能。
- 提供“安装到手机桌面”入口。

### 6. 安装应用页面

按照 `mobile_install_app.png`：

- 检测是否已安装为 standalone。
- 安卓 Chrome 支持 `beforeinstallprompt` 时显示“安装应用”按钮。
- iPhone/iPad Safari 无法直接触发安装时，显示清楚步骤：
  1. 点击浏览器分享按钮
  2. 选择“添加到主屏幕”
  3. 确认名称“别忘订阅”
- 已安装时不要重复显示安装按钮。

## 五、PWA 必须实现

请根据当前技术栈，以兼容 Cloudflare Workers 部署的方式增加：

1. `manifest.webmanifest` 或等价 manifest。
2. 应用名称：
   - name：`别忘订阅`
   - short_name：`别忘订阅`
3. `display: standalone`
4. 合理设置 `start_url`，登录态和管理路径不能被破坏。
5. 设置主题色和背景色。
6. 提供至少：
   - 192×192 图标
   - 512×512 图标
   - Apple Touch Icon
   - maskable 图标（条件允许时）
7. HTML 增加：
   - manifest link
   - theme-color
   - apple-mobile-web-app-capable
   - apple-mobile-web-app-status-bar-style
   - apple-mobile-web-app-title
8. 注册 Service Worker。
9. Service Worker 只缓存安全的静态资源和应用外壳：
   - 不缓存登录提交、配置保存、Token、API Key、通知测试等敏感请求。
   - API 请求优先走网络。
   - 更新缓存版本时能够清理旧缓存。
10. 增加安装状态检测和安装提示组件。
11. 不要为了“离线”伪造过期订阅数据；离线时应明确显示数据可能不是最新状态。

## 六、响应式和视觉标准

- 风格：简约、轻量、白底、浅灰背景、蓝色主按钮。
- 圆角统一，建议 10 至 14 像素。
- 阴影很轻，不要大面积重阴影。
- 页面间距使用统一的 4/8 像素倍数体系。
- 主色、成功、警告、错误、文字、边框定义为 CSS 变量。
- 保持足够对比度和键盘焦点样式。
- 支持鼠标、键盘和触屏。
- 不要照搬参考图中的假数据；页面继续使用项目真实数据。
- 参考图片决定布局和视觉方向，现有功能决定实际字段和行为。

## 七、测试要求

完成后必须实际运行项目并检查：

1. `/admin`
2. `/admin/config`
3. 添加订阅
4. 编辑订阅
5. 删除或停用订阅
6. 搜索和分类筛选
7. 农历开关
8. 登录和退出
9. 各通知渠道配置显示与测试按钮
10. 360、375、390、412、430、768、1280、1440、1920 像素页面
11. PWA manifest 是否可访问
12. Service Worker 是否成功注册
13. 手机端是否无横向滚动
14. 浏览器控制台是否无明显错误
15. 原有 API 和 Cloudflare 部署是否仍能使用

如果项目有现成测试，运行测试。没有测试时，至少运行 lint、构建命令和关键页面手动检查。
